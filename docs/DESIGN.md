## Theme System (主题系统)

agent-mail 支持 3 个主题切换(详见 [SPEC §3.6](./SPEC.md)):

| ID | 名称 | 调性 | 典型用途 |
|---|---|---|---|
| `protocol-registry` | Protocol Registry | 米黄底 + 黑字 + 终端绿 | **默认**。Cyber-Industrial 主页调性 |
| `terminal` | Terminal | 全黑底 + 终端绿字 | 沉浸式命令行审美,夜间 / 长时间使用 |
| `studio` | Studio | 纯白底 + 黑字 + 中性灰 | 阅读、打印、截图分享 |

### 实现机制

- **CSS Variables**:每个 color token 暴露为 CSS 变量(如 `--surface`, `--primary`)
- **挂载点**:`data-theme="<id>"` 挂在 `<html>` 元素
- **持久化**:localStorage `agent-mail.theme`(详见 SPEC §3.6)
- **切换器 UI**:顶部导航栏,所有页面可见(详见 LAYOUT §3.14)
- **SSR 防闪烁**:服务端读 cookie `theme` 渲染 HTML 属性

### 主题派生原则

- **typography / spacing 跨主题完全一致**(不因主题变化)
- 仅 **color tokens** 差异(每个主题都有完整 ~50 token set)
- 颜色对比度保证 WCAG AA(正文 ≥ 4.5:1)
- 状态色 `error` / `warning` 跨主题保持语义一致

### Token 引用方式

- **CSS**: `var(--surface)` / `var(--primary)` / `var(--tertiary-fixed)`
- **Tailwind**: 通过 `data-[theme=terminal]:bg-[var(--surface)]` 变体匹配
- **JavaScript**: 通过 CSS `getComputedStyle` 读取,避免硬编码 hex

## Brand & Style

This design system is built for "Agent-Mail," a decentralized communication layer. The brand personality is clinical, utilitarian, and high-fidelity, evoking the feeling of a cold-war era industrial operating system or a modern cryptographic terminal. It is designed for technical users who value transparency, density, and structural clarity over decorative fluff.

The aesthetic follows a **Cyber-Industrial** movement. It utilizes high-density information layouts, rigid grid systems, and technical markers inspired by industrial fabrication labels. The interface functions as a "system log," where every element serves a diagnostic or navigational purpose. Visual interest is derived from typography, alignment, and hairline borders rather than shadows or gradients.

> **主题无关性**:本节的品牌定位、气质描述、Cyber-Industrial 调性对 3 个主题都适用。具体色值见下方 ## Colors 与 YAML。

### protocol-registry(默认)的设计语义

- **Background (Base):** A warm, off-white/cream (`#fcf9f3`) that mimics the texture of technical manuals or industrial cardstock.
- **Typography & Ink:** Sharp black (`#0d0d0d` → `on-surface: #1c1c18`) for all primary text and structural borders.
- **Functional Accents:** A singular "Terminal Green" (`#00ff41` / `tertiary-fixed-dim`) is used exclusively for "Active," "Verified," or "Success" states. An "Industrial Amber" (`#ffb800`) is reserved for warnings or low-level system alerts.
- **Dividers:** Hairline strokes in black or a muted neutral-gray to maintain grid structure without visual weight.

### terminal(暗色)的设计语义

- **Background:** 深黑 `#1c1c18`,营造夜间 / 沉浸感
- **Primary:** 终端绿 `#00ff41` 直接作为前景色,系统提示语(primary)与正文同色相
- **Container:** 容器略亮于背景(`#31302d` / `#25241f`),建立层级
- **Outline-variant:** 用 `#444748` 而非 `#c4c7c7`,与暗背景保持适当对比
- **Error:** 提亮至 `#ff6b6b`,暗背景下保证可读性

### studio(浅色)的设计语义

- **Background:** 纯白 `#ffffff`,最大化阅读区对比
- **Surface Container:** 多档浅灰(`#fafafa` / `#f5f5f5` / `#efefef` / `#e8e8e8`),用于卡片层级
- **Primary:** 纯黑 `#000000`,与 protocol-registry 一致
- **Outline-variant:** 保留浅灰 `#c4c7c7`,在白底上柔和不刺眼
- **Warning:** 保持 `#ffb800`,暖琥珀在白底上同样醒目

## Typography

> 跨 3 主题完全一致。

The typography is strictly monospaced to reinforce the terminal aesthetic. All headlines and primary labels are rendered in **Space Mono** for its geometric, technical character. Body text and data tables use **JetBrains Mono** for increased legibility at high densities.

**Rules for use:**
- **Case:** Use Uppercase for all headers, buttons, and system labels to emulate industrial marking.
- **Alignment:** Lean into asymmetrical but intentional alignment. Use "Block" justification for data summaries.
- **Hierarchy:** Established through weight and tracking (letter spacing) rather than large size differentials.

完整 token 见 YAML frontmatter `typography`(7 个 typography variant)。

---

## Layout & Spacing

> 跨 3 主题完全一致。

The layout uses a **Rigid Grid** model based on a 4px baseline. Everything is contained within visible or implied borders, creating a "module" based interface.

- **Grid:** A 12-column desktop grid with 1px black borders (gutters).
- **Density:** High. Information should be packed tightly, separated by thin lines and distinct labels (e.g., `[ SN: 0429-ISL ]`).
- **Adaptation:** On mobile, modules stack vertically, but maintain the "boxed" container look. Margins are reduced to 12px to maximize data horizontal space.
- **Alignment:** Elements should frequently align to the top-left of their respective containers, mimicking a command-line prompt.

完整 token 见 YAML frontmatter `spacing`(5 个 spacing variant)。

## Elevation & Depth

> 跨 3 主题适用;具体色值通过主题 tokens 引用。

This design system avoids shadows entirely. Depth is communicated through **Structural Layering** and **Tonal Inversion**:

- **Tier 1 (Surface):** The theme's `background`(`protocol-registry`: `#fcf9f3` / `terminal`: `#1c1c18` / `studio`: `#ffffff`).
- **Tier 2 (Modules):** 1px outlines (theme `outline` token) define sections.
- **Tier 3 (Active/Hover):** Solid `primary` fills with `on-primary` text.
- **Z-Axis:** Instead of lifting elements with shadows, "raised" elements are represented by thicker borders or "inverted" color blocks (`primary` background with `on-primary` text).

---

## Shapes

> 跨 3 主题适用;与颜色无关。

The shape language is strictly **Sharp (0px roundedness)**.

Every button, input field, and container must have 90-degree corners to reflect an industrial, "machined" aesthetic. Use 45-degree chamfered corners occasionally for specialized status tags or "tab" headers to simulate technical decals.

---

## Components

> 跨 3 主题适用;颜色通过 tokens 引用,不硬编码 hex。

- **Buttons:** Solid `primary` rectangles with `on-primary` text in `label-caps`. On hover, the background changes to `tertiary-fixed` (terminal green). No rounded corners.
- **Input Fields:** Defined by a bottom-border only (`outline-variant`) or a full 1px box (`outline`). Include a permanent "prompt" character (e.g., `>`) before the user text.
- **Chips/Status Tags:** Encased in parentheses or brackets, e.g., `( STATUS: ACTIVE )`.
- **Cards/Modules:** Use a "Header Strip" approach where the title of the module is in a `primary` bar at the top, with the content in a `surface` box below it.
- **Dividers:** Use dashed lines (`----`) or solid 1px lines (`outline`) to separate horizontal data points.
- **System Markers:** Incorporate non-functional glyphs like `/`, `>`, and `_` to decorate the corners of modules, reinforcing the "code" environment.
- **Lists:** Use monospaced indices (e.g., `01.`, `02.`) and ensure vertical alignment of text across list items.
